## ① 问题: 现 arc-select 为何不适合温情向(四处根因, 按优先级)

现行三层数据流: `siftStories`(挑弧)→ `pickArcStart`(REAGAN 排序 + 定 inciting 起笔 tick)→ `longrun.ts:244` 塞进 `arcHint` 注入 ch1。整条链是为**「最大化戏剧张力」**校准的, 温情向四处错配:

- **根因 A(天花板·结构性)— 弧池全冲突**。`siftStories`(`sim-fitness.ts:63-108`)能产出的 pattern 只有 7 种、全部由死亡/复仇/覆灭/雪崩触发。即便把 REAGAN 调平, 温情世界也只能从"谁死了、谁灭门、谁雪恨"里挑起笔。**且温情世界根本不发这些事件 → sift 空集 → 返回 null**(比爽文更糟: 不是挑错弧, 是无弧可挑)。

- **根因 B — REAGAN 把最冲突的弧顶到最高**(`arc-select.ts:10`)。`复仇闭环/崛起陨落:1.2` 并列最高(注释明写依据 EPJ2016"含跌的复杂弧参与度更高")。温情恰要反过来: 重"成长抵达、久别重逢、释怀", 而非"跌得越惨越好"。**参与度代理 ≠ 温情代理。**

- **根因 C — arcHint 硬钉"矛盾/张力"**(`longrun.ts:244`)。风格无关, 与同一 outline prompt 里的 `GENTLE` beatSpec(`:166`)、`PENMANSHIP` 温润(`:37`)**直接打架**: 起笔框喊"一翻开就在矛盾里", 节拍/笔法喊"流连留白、不必生新冲突", 模型在"该不该立刻起冲突"间摇摆, 首章基调撕裂。

- **根因 D — 起笔取点钉引爆点**(`arc-select.ts:22,25`)。`incite = atTick − span` 是 inciting-incident 戏剧理论(合理但为爽文调): 把开篇钉在冲突引爆点前夜。温情向想起笔于"日常/抵达/重逢之前的余韵", 而非引爆点。`spanPen`(`:24`)又把温情最爱的长跨度陪伴弧压掉。

> 优先级: **A(无弧可挑) > C(基调打架) > B(排序偏冲突) > D(起笔钉引爆点)**。
> T1 根治 C, 缓解 B/D; T2 根治 A(扩弧池, 解 sift 空集)、彻底反转 B/D。

## ② 温情起笔该是什么: 从人情时刻/余韵起, 非冲突峰值

### 情感生成机制与爽文相反(理论背书)

- 爽文张力来自**未解的问题**(他会赢吗?谜底?)→ in-medias-res 把读者扔进矛盾、用悬念拉人(这是现算法取 inciting incident 的全部理由)。
- 温情感染力来自两个**状态**(非问题): ①**「已拥有」与「即将失去」的并置**(物哀); ②**「被另一个人看见」的瞬间**(连接)。状态需读者先**安住其中**才能被触动。把读者扔进冲突 = 剥夺温情赖以发生的"安住"。
- 硬背书: **kishōtenketsu(起承转结)** 证"无冲突亦可成章"(戏来自第三幕"转"=意外并置, 非升级对抗); **iyashikei(疗愈系)** 做"情绪平整"、刻意回避剧烈起伏; **understatement(留白, Munro 范式)** 克制放大而非削弱情感; **物哀** = 对消逝的敏感。

> **温情起笔 = 一个让读者愿意慢下来、且立刻感到"这里有温度"的寻常时刻; 它本身没有悬念, 但埋着一个"这会失去"的影子(物哀钩)。**

### 经典温情开篇的共同配方(prior-art: 雪国/厨房/Olive Kitteridge 全部慢起)

1. 入口是**一个具体物件 / 一段日常动作 / 一处移动中的风景**, 不是一个事件。
2. 第一页就**埋无常/失去的情绪底色**, 只暗示不解释。
3. **把平凡神圣化**——细节承载情感重量(是"失去某人后才会留意的那种细节")。

### 起笔时刻 = 情感弧上的"肩部(shoulder)", 不取峰值

四个候选其实是同一条弧的不同切点。**裁决: 双层起笔结构**(微光引子 + 缘起肩部):

```
[第1章 开篇切片]
  ├─ 引子(前 ~1/4 章): 一个【寻常人情时刻】——主角在人间一件小善/一次萍水相逢
  │     作用 = 立刻给"温度" + 立人设(长生者的温柔与疏离) + 零前情门槛
  │     物哀钩 = 这个被善待的人, 主角知道自己会看着他老去 / 这一面也许是最后一面
  └─ 主弧起笔(后 ~3/4 章): 切到【一次相遇 / 一段日常微光】的【平静肩部】
        = 本卷主线羁绊的"缘起"或"正共处的寻常日子"; 读者在这里安住、与主角一起"拥有"这段关系
        ↓ (这段关系的峰值——深缘之极 / 那场告别——是本卷要【建向】的目标, 不在开篇出现)
```

**与现算法的镜像原则(继承, 仅改定义)**: 现算法说"峰值(高潮)是开篇要**建向**的目标, 绝不在它起笔"——温情向**继承这条**, 只把"峰值"从"冲突最炸"换成"**情感最满**(深缘之极 / 那场死别 / 那次救赎完成)"。**告别是温情的高潮, 和爽文的冲突峰值一样不能用来起笔——要建向它。** 开篇就告别, 读者还没爱上, 泪点是空的。

肩部精确定义(回答"挑余韵处起笔"):
- **升前肩(pre-rise shoulder)**: meet 事件后、bond 刚萌芽的第一个平静相处 tick = "相遇之初的暖"。**默认取此**(最稳)。
- **降后肩(post-peak shoulder / 余韵)**: 失去**之后**的第一个平静 tick(葬礼散后、空屋、遗物)= "已失去的静默"。**当本次失去特别有重量时取此**(一上来人去楼空的静, 再倒叙整段暖, 反差出物哀)。
- **绝不取峰值 P 本身**(那是告别现场 = 高潮)。

## ③ 预热: 保留, 但改挑选目标(温情比爽文**更**该预热, 且更久)

### 论证: 温情是预热的最佳受益者(prior-art 两个硬数据的温情重读)

1. **Smallville**: 2 天自由跑, 社交网络密度 0.167→0.74(≈4×)、知晓率 4%→52%。这些数据**主要产出的是"关系连接"本身**(谁认识谁、谁记得谁)。对爽文是副产品, **对温情是主菜**——"互相知晓 + 一起经历过小事 = 故人/旧识/上次你帮过我"的温度底座。
2. **StoryBox**: 先模拟越久, Character Development 越好。温情向全部重量压在 character & relationship 上, **是这条曲线斜率最陡的受益者**(爽文还能靠外部冲突撑场, 温情没有外部冲突可借, 只能靠预热攒的关系厚度)。

### 零起点 vs 预热即有故人——后者赢, 且与长生母题天作之合

| | 零起点快起笔 | 预热后起笔(开篇即有旧识/旧账) |
|---|---|---|
| 关系厚度 | 0, 需正文现攒(前 10 章铺关系, 慢热) | 预热 N tick 已攒, 开篇即"有历史" |
| 物哀强度 | 弱(刚认识的人逝去, 读者无感) | **强**(预热里"看着他从壮年到衰老"的时间真实流过) |
| 温情密度 | 慢热 | 第 1 章就能写"老张啊, 二十年前你也是在这棵树下…" |
| 长生母题 | **冲突**(长生者核心体验是"时间的厚度", 零起点抹掉它) | **天作之合**(预热模拟出的"逝去的时间"正是长生者要承载的) |

> **爽文用预热攒"火药"(恩怨), 温情用预热攒"包浆"(旧识旧账旧物)。一个没有过去的不死者, 不温情。**

### 预热多久 + 怎么用

- 爽文甜点 N≈30–60 tick(攒够一桩冲突)。
- **温情向 N≈60–120 tick**(≈20–40 章世界历史), 甚至**允许预热内某些 NPC 完整老去/逝去一轮**, 让开篇主角的"故人"有重量。
- **过载缓解(对应 pre-evolution 的 168tick≈56章压垮警示)**: 预热出的长历史**不全搬进正文, 只作主角"记忆库 / lore"**; 开篇正文只锚定其中一段关系, 其余历史经回忆/睹物思人**渐进渗出**(正好是温情最爱的手法)。**预热长 ≠ 叙述长**; 温情把"过长预热历史"从爽文的"过载风险"转成"记忆的纵深"。

### 挑什么弧(本节的核心裁决)

**保留预热管线与两阶段骨架, 但把 scout 的挑选目标从"最有戏的冲突弧"换成"羁绊最厚 × 速朽性最强 × 物哀形状 × 寻常可入"的关系弧**(详见 ④)。T1 在现有冲突弧里"捞人情"(路线B, 缓解), T2 扩关系型 pattern(根治 sift 空集)。

> 诚实边界: Smallville/StoryBox 的"关系"是社交图连通性 / 通用 character development, **不等于"温情向的羁绊质量"**。预热能保证"他们认识、一起经历过事", **不能保证"这段关系动人"**——"动人"靠 ④ 的关系型 sift 去**挑** + continuous-minds 的 bond/八字四轴去**填**。**预热是必要不充分条件。** N≈120 是从"长生母题需可感的时间厚度"推的, **需 A/B**。

## ④ 温情弧排序: REAGAN 温情版 + 人情型 patterns + favor 化解结善(三者都做)

现行: `弧线得分 = quality(完整度×跨度×罕见度) × REAGAN形状先验 × 可空降性(spanPen×incitePen)`(冲突校准)。
温情版三处替换:

```
温情弧得分 = 羁绊势能 × (1 + 速朽溢价) × 物哀形状先验 × 寻常可入性
           ──────────   ────────────   ─────────────   ──────────
           取代结构冲突分  取代StU罕见度   取代Reagan跌宕   温情独有
```

### 4.1 REAGAN 温情版(翻转形状先验)

Reagan 6 弧按"情感落点暖意 + 形状柔和度"重排(非戏剧张力):

| Reagan 弧 | 形状 | 温情适配 | 理由 |
|---|---|---|---|
| **Man-in-hole**(跌-起·温柔版) | 跌后回暖 | **顶权** | "失而复得/久别重逢/误会冰释"。Vonnegut: 人物结尾比开篇略高 → 读者得慰藉 = hopeful 内核 |
| **Rags-to-riches**(平缓上升) | 单调上升 | **高权**(与现系统相反) | 关系/心境的缓慢回暖, 靠细微触动累积。现系统压到 0.85 是冲突逻辑, 温情要抬权 |
| **Cinderella**(起-跌-起) | 起跌起 | **中性偏正** | 若那个"跌"是温柔离合而非毁灭, 是合格温情圆满弧 |
| **Icarus**(起-跌) | 起后坠 | **低权** | 仅"物哀抒情挽歌"(盛极而逝)窄口对路, 落点淡哀非创伤 |
| **Oedipus**(跌-起-跌) | 跌起跌 | **最低** | 纯悲剧/复仇结算(现系统顶权的"崛起陨落"即此族), 明确压底 |
| **Riches-to-rags**(单调下降) | 单调下坠 | **次低** | 仅极克制的"凋零美学"偶用, 默认压权 |

> 注: 物哀美学里 fall(纯跌)是**正形**(川端式), 与爽文先验几乎相反(爽文压纯跌=太丧, 温情拥抱纯跌)。但需产品定位对齐: `GENTLE` 应理解为**"物哀向"**; 若要"治愈系大团圆", 形状先验偏 fall-rise(失而复得/送别后释怀), 这是**两种不同温情**。本蓝图默认物哀向(顶权给 man-in-hole 的"温柔低谷+回暖", 兼顾两者)。

### 4.2 人情型 patterns(T2 新增, 与冲突型并行, 不删旧的)

按 story-sifting 形式(有序事件子序列 + 共享角色绑定 + 时序约束), 用**现有遥测**(bond / 恩怨释怀 / MemoryRecorded)定义:

| 人情 pattern | sift 规则(?A=长生主角, ?B=凡人; 全用现有事件) | 情感内核 | 对应 Reagan |
|---|---|---|---|
| **深缘结成** | 同一对 `bond:<id>` 经 ≥4 次 `论道结善` 单调升过阈值, 无中途反目 | "我们成了彼此的人" | rags-to-riches |
| **生死相托** | A 对 B `bond` 高 + B 遭 `CharacterFell`/危局 → A 护持/承志(actCount↑且 bond 维持) | "共渡难关后情更深" | man-in-hole |
| **恩义两清** | `VengeanceResolved` 且 `outcome="恩怨释怀"`(**直接捞 `world-actor.ts:448`**)→ 怨转平/化敌 | "积怨→释怀回暖" | man-in-hole |
| **重逢** | `bond(A,B)` → 分离 → 大 Δt 后 re-meet | "多年后你还认得我" | man-in-hole |
| **传承** | `bond(A,B)` → `death(B)` → A 承 B 之物/愿 | "你留下的我替你看着" | rags-to-riches 长尾 |
| **陪伴终老** | 一对 `bond` 长期高位、跨大跨度共存到末段(长 span 不衰减), 叠 `MemoryRecorded` | "最后一程有人在" | rags-to-riches + 平台 |

> 叙事学锚点: **Hogan《Affective Narratology》** 把 attachment(依恋)/ romantic union / sacrifice 列为与 revenge / criminal-justice **并列但性质相反**的 emotion-driven 普世原型。生死相托=sacrifice, 深缘/陪伴=attachment, 恩义两清=restorative 收束(化解>覆灭)。**人情型 patterns 不是临时凑的, 有跨文化叙事普遍性背书。**
> 对长生主角, **farewell/vigil/legacy 是重复发生的**(一次次送别)——可加权"主角已经历过 m 次离别"抬高"时间厚度"感(这本身就是母题)。

### 4.3 favor 化解结善(对现冲突 patterns 反向加权, T1 即可做, low-risk)

即使不全套新建, 至少把现 patterns 的**结算方向**翻过来(温情 profile 下):
- `复仇闭环`: 现奖 `雪恨`(`sim-fitness.ts:79`)→ 改奖 `恩怨释怀`(化解), `雪恨` 反压权。
- `覆灭复兴` vs `派系覆灭`: 只保留"复兴", 压掉"覆灭"。
- `崛起陨落`(=Oedipus): 压到最低或剔除。
- `factionTension.directness`(正面交锋, `:149-153` 现为正分)→ 负 bond 越多越扣分, 奖正 bond 密度(harmony)。
- `volatility`(`:170-171` 兴亡密度现为正分)→ 翻成惩罚, 奖"平静的连续性"。
