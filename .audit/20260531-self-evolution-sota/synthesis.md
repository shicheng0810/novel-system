# 自进化系统 · SOTA 对比与借鉴蓝图

> 4 路并行调研综合（LLM-进化搜索 / 质量-多样性·开放式 / 自我修改 agent / 经验记忆·自我批评）。
> 目的：把我们「小说自进化」从「单适应度 + 单点爬山」升级到学术/工业界更强的范式。

## 0. 先澄清「EvoMap」

**没有名为 EvoMap 的标准算法。** 两种可能：
- `evomap.ai` / `github.com/EvoMap` 是一个社区维护的 **"awesome-self-evolving-agents" 资源清单**（目录，不是方法，别当方法引用）。
- 几乎肯定是 **MAP-Elites** 的字面简称（"Evolutionary MAP of Elites"）。真正要学的就是 **MAP-Elites / 质量-多样性(QD)**，及其写作版 **QDAIF**（见 §3-A）。

## 1. 诊断：我们现在的位置与病根

自进化领域常用「四阶段循环」对照（综述 arXiv:2404.14387）：**经验获取 → 经验提炼 → 更新 → 评估**。我们的覆盖：

| 阶段 | 我们现状 | 缺口 |
|---|---|---|
| 经验获取 | 每 8 章取最近章节 | 只取最近、无配对（高分 vs 低分） |
| 经验提炼 | 评委吐 overused/wins/fixes | 偏主观印象，无结构化触发条件 |
| 更新 | 单点爬山 4 个旋钮 + 账本注入 | 固定旋钮轮换、无种群、无多样性存档 |
| 评估 | **单一 LLM 评委 → 单一加权适应度标量** | 易被 game、噪声大、信用分配差 |

**核心病根（四路一致结论）**：**单一标量适应度 + 单点爬山 → 必然向「评委最爱的那一种风格」收敛 = 模式坍塌(mode collapse)。** 而小说要的恰是长期多样与持续新意。其余短板：固定旋钮（非 LLM 自由提议变异）、账本全量注入（无情境检索）、评委既当裁判又易被讨好。

## 2. 对比表：SOTA 原理 vs 我们

| 维度 | SOTA 做法（代表系统） | 我们现状 | 差距 |
|---|---|---|---|
| 优化目标 | 多样性存档「照亮」(MAP-Elites / QDAIF) | 单一最优 | ★★★ 坍塌风险 |
| 变异算子 | LLM 自由提议改动 (Eureka/Promptbreeder) | 固定旋钮轮换 | ★★★ 搜索面窄 |
| 反馈粒度 | 分项文字反思 (Eureka reward-reflection) | 单标量 | ★★ 信用分配差 |
| 解的存储 | program database + 血统 + 开放分支 (AlphaEvolve/DGM) | 单基因、退则丢 | ★★ 不可累积/回溯 |
| 防套路 | Novelty Search / PATA-EC | 仅"禁近 6 章雷同" | ★★ 缺主动新颖压力 |
| 记忆检索 | importance×recency×relevance + 技能库 (Generative Agents/Voyager) | 全量塞最近 N 条 | ★★ 召回粗 |
| 记忆维护 | ADD/EDIT/UPVOTE/DOWNVOTE 投票 (ExpeL) | 年龄衰减 | ★ 可更聪明 |
| 进化对象 | 提示模板/rubric/架构 (ADAS/DGM) | 只进化参数 | ★★ 天花板低 |
| 评估稳健 | 客观指标 + 评委解耦 + F/R 分离 (FunSearch/AlphaEvolve/Eureka) | 单 LLM 评委 | ★★★ 可被 game |

## 3. 借鉴蓝图（按 ROI 分层，已映射到小说场景）

### Tier 1 — 立刻止血，最高优先级

**A. 用 MAP-Elites / QDAIF 存档替换「单一适应度」** ★最关键
- 借：QDAIF（Quality-Diversity through AI Feedback, ICLR 2024, arXiv:2310.13032）——**专为创意写作设计的 QD**，LLM 既当变异算子又当"归格+质量"裁判；pyribs 官方有写故事 tutorial，几乎是现成蓝本。
- 映射：定义 4–5 个**行为描述子**作 archive 的轴（彼此正交、对读感敏感）：① 语气基调(冷硬/诙谐/悲怆/热血) ② 叙事视角(第一/第三·限知/全知) ③ 冲突类型(权谋/战斗/情感/解谜/生存) ④ **句长与节奏分布**(平均句长+方差，**纯统计、不用 LLM**) ⑤ 情绪走向(扬/抑/反转)。每维分 3–4 箱 → 网格。**每格存「该风格下 rubric 最高的一章 + 当时的基因 + 那套避雷/发扬/修正指引」。** 质量分从"全局唯一标量"降级为"格内局部竞争"——质量与多样性就此解耦。
- 收益：坍塌从机制上被堵死；额外白得一个"风格调色盘"，可按卷/章点用不同格的基因。

**B. 混合评估：客观指标 + LLM 评委，且客观指标不进生成 prompt** ★防 game
- 借：FunSearch/AlphaEvolve 的可执行客观评分；Eureka 的 **F/R 分离**（最终适应度 F 不直接喂给被进化对象，天然抗刷分）。
- 映射：加一组**确定性程序指标**——短语重复率/避雷命中数、对白占比、章末钩子正则、实体连贯校验、词汇多样性 TTR——与 LLM 评委加权融合。**关键：这些客观指标只用于打分/选择，绝不写进生成提示**（否则生成器直接刷指标）。
- 收益：抗噪、抗 game、评委成本下降。

**C. 标量 → 分项反思反馈** 
- 借：Eureka reward-reflection。映射：保留单标量做选择，但把 rubric 六维分项 + 章节级证据（哪段对白拖、哪个钩子失效）汇成**文字**回传给变异环节。
- 收益：解决信用分配——知道"为什么降分"才能精准改。

### Tier 2 — 搜索能力升级

**D. LLM 提议变异，取代固定旋钮轮换**（Eureka/Promptbreeder 自指）。让"变异 LLM"读最近适应度+账本，自由提议 1–N 个具体改动（改某条避雷规则/调钩子密度/动作权重函数形/采样参数），不再局限预设旋钮。Promptbreeder 还进化"如何变异"的元层。

**E. 解数据库 + 血统 + 开放分支**（AlphaEvolve 程序数据库 + Darwin Gödel Machine, arXiv:2505.22954）。每代基因存成 `(配置→适应度→反思)` 节点 + 父指针血统树；下一代**按分数近似比例、但任意历史节点非零概率**被选为分支起点 → 允许"评分一度下滑的垫脚石"几代后超越全部前代。配 OPRO 式：prompt 里回填 2–3 个历史高分配置当 few-shot。

**F. Novelty 守门防套路**（Lehman & Stanley "Abandoning Objectives"）。为每章算新颖度 = 其 descriptor/嵌入到最近 K 章的平均距离；**即便 rubric 高分，但与近 K 章太像 → 扣分/拒用**，逼新桥段；设质量下界防"为新而烂"。跨世界可用 Enhanced-POET 的 **PATA-EC**（比"一组固定探针在该世界产出的质量排名序"而非比辞藻长相）。

**G. 账本升级：技能库 + 智能检索 + 投票维护**
- 发扬表 → **Voyager 式写作技能库**（arXiv:2305.16291）：每条做成 `{技能名, 触发情境标签, 描述, 微范例, 组合关系}`，按 embedding 检索 **top-3~5** 注入，而非全量塞。
- 检索 → **Generative Agents** 三因子 `importance×recency×relevance`（recency 按"上次被命中"计时，高频有用条目不因年龄掉队）。
- 维护 → **ExpeL** 的 ADD/EDIT/UPVOTE/DOWNVOTE + 重要度计数（初始2，命中+1，失效-1，归零删）替代纯年龄衰减；近似条目 EDIT 合并防膨胀。
- 蒸馏 → ExpeL 配对法：高分章 vs 低分章配对，逼出"具体做错了什么→正确做法"，强制模板 `{触发条件, 该做X, 别做Y, 证据章号}`，杜绝"要写得更生动"这类废话。

### Tier 3 — 更激进（第二阶段）

**H. 进化「提示模板 / rubric」本身**（ADAS, arXiv:2408.08435 + DGM）。把生成 prompt 拆成可寻址槽位（开篇钩子/节奏/对白密度/章末悬念），让 meta-prompt-agent 读评分日志提议一处模板改写，**只有 ≥N 章显著超父模板才进档，否则回滚**；永远保留一个"冻结安全基线模板"作回退锚点。

**I. POET 式「世界设定 ⇄ 文风」协同进化 + 跨世界迁移**（直接吃我们的多世界架构）。把世界设定当"环境 E"、基因+账本当"解 A"协同进化；**关键是迁移**——修仙世界某格淬炼的优秀基因，直接/微调后丢到其它题材世界的同型格子去试，读感更好就采纳。配最小准则：某世界设定让所有基因都写崩(太难)或毫无张力(太易)就淘汰/变异。让四个世界从各自为战变成互相供给垫脚石的生态。

## 4. 防自欺红线（anti-reward-hacking，务必同步落地）

Self-Rewarding LM（arXiv:2401.10020）实证：同模型既生成又评测 → **回答长度 1092→2552 暴涨**式刷分。三道闸：
1. **评委与生成器解耦**：评委用不同模型/不同 system prompt/不同温度，绝不与生成器共享被进化的 prompt。
2. **人工锚定校验集**：留一小批人工标注的金标准章节，每代检测"评委是否还和人类一致"，一致性下降即冻结/回滚评委进化（防 rubric 自我放水）。量不必大，关键是**外部、不可被系统自我优化**。
3. **反代理指标守门**：监控被钻空子的伪信号（章节长度、辞藻堆砌、关键词复读率、"伏笔开了不收"），任一异常飙升即判罚/丢弃该基因。

## 5. 最小落地路径

- **Phase 1（止血）**：A（把现有单"genome"扩成多格 archive，描述子=能算的统计 + DeepSeek 归格）+ B（混合评估）。立刻堵住坍塌。
- **Phase 2（记忆）**：G（技能库 + 三因子检索 + 投票维护）+ C（分项反思）。
- **Phase 3（搜索）**：D（LLM 提议变异）+ E（血统 archive）+ F（novelty 守门）。
- **Phase 4（激进）**：H（进化模板/rubric）+ I（多世界协同迁移）。
- **贯穿全程**：§4 防自欺三道闸；KPI 从"单适应度最大化"换成 **QD-Score（覆盖格数 × 各格质量和）+ 新颖度产出率**。
