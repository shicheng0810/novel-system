## 二、各症 edit-pass 为何没逮住(对照现有 7 维 lint)

现行 `lintChapter`(`edit-ledger.ts:40-66`)只有 7 维:`similePerK / cliches / bodyRepeats / crossImgs / noResp / workingObj / explainTail`。这 7 维全是"语言表层的『过』"(喻/絮/象征/点破),**没有一个测"结构性均质"**——这正是缺口。

| 症 | 最近的现有维度 | 为何漏 |
|---|---|---|
| **①** | `workingObj`(同物≥4次报)| (a)词库 `STATIC_IMG`(`:30`)=[冰棱,竹屑,松针,铜铃,井绳,腌萝卜,灯骨,竹牌,炊烟,落叶,银杏,水痕,豁口,草药,药囊],**碗/灶/姜汤/橹一个都不在** → 碗45/灶24 命中 0;(b)即便逮到,它管"单物扛主题",**不管"每段端新器物缓冲情绪"的章内弥散分布**。`crossImgs` 只做**跨章**(`u.ch>=n-5`)去重,管不了章内。无"器物密度/百字""段尾缓冲比"维度。|
| **②** | 无 | `lintChapter` **完全没有句式/节律维度**。`BODY_SIGNAL`(`:29`)含"手停/手一顿"但当"情绪身体信号一情一信号"查(`>=2`报),只盖"手停",盖不了"碗停/火光缩/水声远"整族。`passesGuards` 只管长度/对话密度/硬词,无任何节奏检测。|
| **③** | 无(设计性推给 persona)| `master-benchmark/synthesis.md:181` 明写"声口同腔不进章级 pass 主战场…交 persona digest"。**但 `persona.ts` 的 `personaDigest`(`:13-29`)只携属性/心境/恩怨(程归——心有所系;牵系:亲廿四),零 speech-level 指纹**(无语速/句长/口吻/抢话)。于是 persona 让角色"带恩怨出场",**台词仍由同一 LLM 用同一温润腔生成**——声纹无源可注。pass 这边连兜底机检都还没实现。|
| **④** | 无 | `lintChapter` 无"详略/推进密度/过细动作"维度。`PENMANSHIP`(`longrun.ts:42`)有"疏密相间"但是**生成端软约束(会回弹,ch231 实测没生效)**。更糟:减法 `buildRevisePrompt` 铁律②③(`:76`)"以动作代抒情/以物代情"是**必须保住项**,等于**鼓励**多写动作器物——pass 结构上**不会**去删微动作,与症④正相反。|

**关键架构铁证(致命断点):** `personaBlock`(`persona.ts:32`)→ `bibleEcho`(`longrun.ts:429`)→ 作 `bible` 参传入 `writeChapter`,但在 `writeChapter` 内 `bible` **只进 outline 提示(`:207`)**;真正逐段写台词的 `secPrompt`(`:228`)用的是 `scene/ros/prev`,**persona 的声口信息从未到达写对白的那一步**。这从架构上解释了症③为何无解。

## 三、修法清单(按症)

> **二分处置**(沿用 master-benchmark 框架):**症①②④ = pass 可补的"过"**(扩机检 + 减法指令);**症③ = persona 主战场**(声口指纹机制,见第四节,pass 只兜底)。全部 GENTLE-gated。
> **阈值已按好章校准**:只逮病例、不误杀 renjian/shanju 好章。

### 修法 A · 症①:器物缓冲(段尾环境收尾)— `edit-ledger.ts`

**机理**:不计器物总数(噪声+反罚回扣),改测**"叙述段以环境白描+一个微动收束、把情绪缓冲掉"的结构模式**。
**改哪行**:
1. `:30` 后新增**世界无关**常量(勿污染 `STATIC_IMG`):
   ```ts
   const SETTLE_END = /(暗了|又暗|垂.{0,2}回去|停住|稳稳.{0,2}停|不动了?|没.{0,2}动|远了|散了|落下来?|洇开|淌下去|水渍|发白|碎光|聚拢|往里鼓|一响|响了一下|簌簌|渐.{0,3}息|尾音|霜)/;
   ```
2. `:51` 区(`workingObj` 旁)新增计数(**tail-1/3 匹配**,非严格行尾——病例段尾缓冲多在段中,严格 `$` 只命中 10%,tail 法可升到 ~30%、好章仍 ≤5%):
   ```ts
   const LQ = "“";
   const narrParas = paras.filter((p) => !p.includes(LQ));
   const settle = narrParas.filter((p) => SETTLE_END.test(p.slice(-Math.max(12, p.length / 3 | 0)))).length;
   const settleRatio = +(settle / (narrParas.length || 1)).toFixed(2);
   ```
3. `:62` 后新增触发(**阈值 ≥0.20**,病例 ~0.30~0.37 / 好章 ≤0.05):
   ```ts
   if (settleRatio >= 0.20) issues.push(`器物缓冲成例(${(settleRatio*100|0)}% 叙述段都以"环境白描+一个微动(暗了/远了/垂回去/一响)"收尾,把情绪缓冲掉——成机械套路): 删掉至少一半这类段末缓冲,让部分段直接收在人物动作或话里;器物全章克制反复回扣一两件(那只碗/那截甘草),不必每段端出新物。`);
   ```
- **GENTLE 安全**:`lintChapter` 仅 `EDIT_PASS`(`longrun.ts:45`,`GENTLE && NOVEL_EDIT_PASS!=="0"`)路径调,天然 gated。
- **爽文逐字节**:爽文 `EDIT_PASS=false`,`lintChapter`/`reviseChapter` 永不入路径 = 零变更。
- **风险**:低。删段末缓冲句、不删器物本身/不删对话。
- **三道闸误杀评估**:病例 70 叙述段删一半缓冲 ≈ 13 段尾句 × ~10字 ≈ 130字 = 5007 的 **2.6%**,远不触 `passesGuards` 长度地板 0.85(要删 750字才触)。对话密度地板不动(删的是叙述)。**安全。**
- **大师 craft 映射**:汪曾祺《葡萄月令》多数名词素面朝天("葡萄睡在铺着白雪的窖里"),器物只在承载关系那一刻出现一次;萧红式"差异中深化"(我家是荒凉的/院子是荒凉的)用同一意象**深化**而非每段当缓冲垫。

### 修法 B · 症②:句式节律(停顿拍 + 段落模板均一)— `edit-ledger.ts`

**机理**:测"主语+凝滞动词"停顿拍频率 + 段尾停顿凝滞比例(近似"动作→环境→对话→停顿"四联模板均一度),而非整体句长方差(无判别力)。
**改哪行**:
1. `:31` 后新增:
   ```ts
   const PAUSE_BEAT = /(手|火光|水声|声音?|脚步|碗|汤|目光|火|帘子?|指节|喉结|嘴角)[^。，！？“”]{0,7}(停|顿|缩|远了又?近?|不动|没动)/g;
   const PAUSE_END = /(停|顿|缩了|远了|不动|没动|静|沉默|一响|渐.{0,3}息)$/;
   ```
2. `:51` 区新增计数:
   ```ts
   const pauseBeats = (body.match(PAUSE_BEAT) || []).length;
   const pauseEnd = narrParas.filter((p) => PAUSE_END.test(p.slice(-8))).length;
   const pauseEndRatio = +(pauseEnd / (narrParas.length || 1)).toFixed(2);
   ```
3. `:62` 后新增触发(停顿拍 ≥6 病例 8;段尾停顿比 ≥0.06 病例 9% / 好章 1~3%):
   ```ts
   if (pauseBeats >= 6) issues.push(`停顿拍反复(全章"X停了/顿了/缩了/远了"类凝滞收束≈${pauseBeats}次,有气氛但重复后机械): 保留最有分量的两三处,其余删或改为继续动作;让节奏不规整、容得下突兀的短句、不收在停顿上的段;不必每个情绪都用"手停/火光缩/声远了"镇一下。`);
   if (pauseEndRatio >= 0.06) issues.push(`段落节律太规整(段段近乎"动作→环境→对话→一个停顿"同一模板): 打破规整——有的段用一句短促的话收,有的段一笔带过不停顿,让长短句错落、有人写长篇的不规则瑕疵感。`);
   ```
- **GENTLE 安全 / 爽文逐字节**:同 A(`EDIT_PASS` gated,爽文不入路径)。
- **风险**:中。停顿/留白是温润命门、绝不能清零→指令只"削峰到 2-3 处"不禁用;LLM 可能删太狠致语气干瘪。
- **三道闸误杀评估**:删停顿拍多删短小句、单删字数极少(每处 6-8字),不破长度地板;`passesGuards` 对话密度地板(±2.5/k)挡住"把停顿全改成对话"的越界。**护栏**:减法措辞写"保留最有分量的两三处"+ 铁律④"留白式收尾"+ "拿不准就少删"兜底。**安全。**
- **大师 craft 映射**:孙犁《荷花淀》最紧张处插从容荷花;汪曾祺《受戒》前慢后爆、《葡萄月令》"一月,下大雪。雪静静地下着。果园一片白。"全短句砸节奏——**疏密相济、节律不可预测**,非 ch231 均长 27 字段段同模板。

## 四、声口分化机制(症③治本)

**目标**:不贴口头禅(伪差异比同腔更假),而是给每角色一个**稳定的句法/语速/态度声口指纹**,从既有符号态**确定性派生**(零新 LLM、resume 安全),随 persona 进生成端。

### 4.1 指纹来源(从现有 props 确定性映射)

> 实证:`persona.ts` 的 props 里**无八字/日主/十神字段**(grep 确认),但有 `innerDrive`、`历练`、`bond:*`、`avenge`——这三样足够派生 4 轴。八字四轴(参考 `novel-continuous-minds` 的 appraisal 写回 stress/bond)可作未来增强,当前用 innerDrive+历练+bond。

**4 轴(派生表 ~7 条,放 `persona.ts` 常量、与 `INNER_CN` 并列):**
1. **语速/句长** ← `历练`:`≥15`(历经百劫)→话疏、短、留半句;`<6`(年轻)→话密、爱补充来龙去脉。
2. **态度/句式** ← `innerDrive`:`ambition/vengeance`→直、断、少寒暄;`attachment/want`→迂、爱反问试探;`grudge`→冷、短、刺一句;`calm`→慢、稳、答非所问。
3. **疑问/软化倾向** ← 年轻+`attachment`→多用"么/呢/吧"和反问;沉稳长者→平陈句、少助词。
4. **抢话/接话节奏** ← 话密角色"会抢、插话、答非所问带出自己的事";话疏角色"短答+停顿"。
5. **身份行话(可选叠加)** ← canon attrs:采药人爱拿草木打比方、掌柜带算账口吻、船工带水上话。

**产出示例**(ch231 三人):
- 廿四(年少/attachment)→ "声口:年少话密,爱把来龙去脉说全,带一点试探的反问"
- 程归(历练深/心有所系)→ "声口:历练深、话疏而短,常留半句,少寒暄"
- 姜未(腼腆少年)→ "声口:先答事实再慢补一句,不主动展开"

### 4.2 注入点(治本关键 = 修复架构断点)

**断点回顾**:persona 现只进 outline(`longrun.ts:207`),**没进写台词的 `secPrompt`(`:228`)**。两处都要接:
1. **`persona.ts:13` `personaDigest`** 加 `gentle` 形参(默认 false),`true` 时末尾追加 `；说话：${voiceTag}`。**爽文调用不传 → 不拼声口 = 爽文零变更**。
2. **`persona.ts:43` `personaBlock`** 标题改为"【在场者·各怀心事**与声口**(对白须随各人声气变,勿千人一腔)】"。
3. **`longrun.ts:228` secPrompt** 在 `${PENMANSHIP}` 后注入在场者声口卡(≤4 人控长度):

- **GENTLE gated**:`gentle` 形参 + `GENTLE &&` 注入双重 gate。

- **红线自查**:声口标签是**句法/语速/态度倾向(话密/话疏/爱反问/直/迂),不是固定口头禅词**(master-benchmark 明确警告"别给角色贴口头禅")。
