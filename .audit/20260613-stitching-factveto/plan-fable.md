# factC 治本方案 · Fable 评审回带(veto→grounding 重框)
> 2026-06-14 · 回应 HANDOFF-to-fable.md 的"怎么真正降下 factC 主体"。Phase 1a 已证: 确定性 veto 在数值/语义类无甜点(同 D20 墙)。本方案给出绕墙路径 + 分阶段执行。
> 同纪律: 喂数据非散文 · judge/抽取≠写手(解耦模型) · GENTLE-gated 默认关 · 四道闸 · 不碰活世界 · A/B fork 中段(冷启太干净) · 先离线证精度再接生成端。

## 0. 核心重框(为何 Phase 1a 卡住、出路在哪)
- Phase 1a 做的是**检测矛盾**(段写完→比对→否决)。而"真矛盾 vs 合法变化(消耗/变价/移动)"的判别本质是 zero-FP 不可能题(= D20/lint-seams 同墙·需 LLM)。
- **治本不必检测, 改做接地(grounding)**: 把**已立微事实当 ground-truth 数据**钉进后续段 prompt, 让后续段**没机会编新值**(数据注入·非 v3 软劝告)。
- 接地的错误剖面**友好**: 抽到错事实→锁错值(有害·但对"干净可抽"的事实罕见); **漏抽→该段不接地·退回现状(无害)**。对比 veto: 误检→无谓重生+改坏好文(有害且贵)。
- 关键: **Phase 1a 的"0 真阳"不挡接地**——接地不需要逮矛盾, 只需抽事实; 它已证的"称谓法名 0 误检抽取"对接地是**现成可用**的干净料。

## 1. 三阶段(measure-first·先便宜安全·再贵)

### Phase 1b【先做·零新 LLM 成本·零误锁风险】确定性接地(清洁类)
把 Phase 1a 已建、已证 0 误检的**称谓/法名/性别抽取**, 从 veto **翻成 grounding**:
- 段 i 写完 → 确定性抽已立 `师太=静檀 / 林思齐=男` → **追加进章内 ground-truth 账** → 段 i+1..N prompt 顶部钉 `【本章既定·必用此值·勿另立】师太=静檀；林思齐=男`。
- 数据型(entity→value·非散文)→ 不触 induction; 清洁类→ 锁错风险≈0; 无 veto → 无重生、无烧 token。
- **验证接地机制本身**(这是 1b 的真目的): A/B fork 中段(两臂 pro)·主轴 = 判官 factC 的**专名/性别子类** A vs B(目标显著降); 副 = d1c(应几乎不动·注入极小)、章长、判官其余类。
- 通过 = **接地机制在清洁类被证有效** → 落地(它本就是 HANDOFF 选项 a 的"小片", 但以 grounding 形态·比 veto 更安全)→ 且**给 Phase 2 的贵抽取吃下定心丸**(机制半边已验)。

### Phase 2【主体·付 LLM 抽取·先证精度】LLM 抽取 → 运行式 ground-truth 账
1b 证机制后, 把抽取从清洁类**扩到数值/语义主体**(确定性抽不了的那 ~70%):
- 段 i 写完 → **解耦 LLM**(≠写手·可用便宜模型/CoVe 式独立角色)抽**本段新立全部微事实** `entity→value`(船资/绳长/凝皮数、关系来源、在场) → 追加进 ground-truth 账 → 钉死喂段 i+1..N。
- 这是评审推荐的"段1抽→锁喂2-4"的**运行式泛化**(每段都抽都喂·覆盖涌现微事实·非只段1)。是唯一能碰主体的路(确定性已证不行)。
- **先离线证抽取精度**(铁律·别拿生成 token 试错): 拿解耦 LLM 在 72 章判官语料上抽事实 → 人工/判官核**抽取准确率**(抽错=锁错=有害)。门槛: 抽错率低到可接受(漏抽无害·只看抽错)。**过了才接生成端。**
- A/B fork 中段(两臂 pro): 主 = 判官 factC 主体 A≤B×0.6(腰斩); 副 = d1c 不炸(数据注入应 benign·须证)、章长不催肥、**抽取 LLM 调用数/章**(成本遥测)。
- 成本: +约 3 次解耦抽取/章(段间)。factC 是 pro/flash 共犯的系统命门·治完两模型都受益且解锁 flash 复测·值这个成本——**但由 A/B 数据 + 你人签定**。

### Phase 3【可选·安全网·先量残差】LLM 矛盾否决(veto)
- 仅当 Phase 2 接地后**判官仍报残余 factC**(接地没覆盖到的)才上。
- 段后解耦 LLM 比对 ground-truth 账判硬矛盾 → 命中则重生(锁值·封顶2·接受+flag)。即 HANDOFF 选项 b, 但**降级为 Phase 2 的补漏**而非主力(因为预防比补救便宜安全)。
- 大概率 Phase 2 接地做对了就不需要——**先量残差再决定**, 别预先建。

## 2. 投资优先级
| 阶段 | 做什么 | 成本/风险 | 何时 |
|---|---|---|---|
| **1b 必做** | 清洁类确定性**接地**(复用 1a 抽取) | 零新 LLM·零误锁 | 即刻·证接地机制 |
| **2 主攻** | 解耦 LLM 抽取→运行 ground-truth 账(治主体) | +3 抽取/章·抽错风险 | 1b 证机制 + 离线证抽取精度后 |
| 3 可选 | LLM 矛盾 veto 补漏 | 重生烧 token | 仅当 2 后有残差 |
| 永久不做 | FULLCTX / 同模型 rewrite / 确定性数值 veto(1a 已证无甜点) | — | — |

## 3. 为何这套对(谱系印证)
- **veto→grounding** 绕开 zero-FP 不可能题(D20/1a 墙): 预防不需要判别"真矛盾 vs 合法变化"。
- **数据非散文**: P1 v2 已证数据注入有效且不炸套语(对比 FULLCTX 散文→induction 爆)。
- **结构化状态做唯一真相源**: 研究 Phase1(DOME 冲突减87.61% / StoryState)正是此路·本方案=把它做进章内段间。
- **judge/抽取≠写手 + 先离线证精度**: 防自偏 + 防拿生成 token 试错(研究"抽取误差=第一风险"·故 Phase 2 门槛只卡抽错率)。
