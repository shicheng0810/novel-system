# 世界模拟器自进化 — 研究综合 (模拟层, 非作者层)

> 4 个并行研究 agent 的综合: ALife/开放式模拟进化 · 生成式智能体社会 · 进化模拟规则(元层) · 涌现叙事的趣味度量与导演。
> 目的: 不只让"模拟写成章节"更好(作者层, 已做), 而是让**世界本身的演化动力**(角色决策、派系消长、事件触发、命运交织)自动更丰富、更有戏、更涌现。

---

## 0. 框定: 我们现在在哪

| 层 | 进化的对象 | 现状 |
|---|---|---|
| **作者层** (已建) | 怎么把模拟写成章节: 文笔/采样参数、避雷、MAP-Elites 风格存档、铁律(叙事)、canon 一致性 | ✅ v2 MAP-Elites + 全局迁移 + 规则层 + canon, 已激活 |
| **模拟层** (本研究) | 世界本身怎么演化: 角色怎么决策、派系怎么消长、事件怎么触发、命运怎么交织 | ❌ 只有 **1 个旋钮**(八字/奇门引导强度 priorWeight)在进化, 其余动力全是写死常数 |

**核心结论**: 模拟层自进化 = 4 个齿轮咬合 —— **更丰富的涌现底座 → 可验证的"有趣"度量 → 规则/参数自进化引擎 → 混沌边缘的自我调节**。缺任何一个都跑不起来。下面逐一给出, 并映射到我们的真实代码。

---

## 1. 最大的拦路虎: fitness ("一个模拟世界够不够有趣"怎么自动测?)

这是模拟层进化的命门 —— 作者层有 LLM rubric + 客观文本指标当 fitness, 模拟层没有。4 份研究**高度收敛**到一套答案, 且都强调"必须是可验证的符号指标, 否则进化会去骗裁判":

### 1.1 主信号: Story Sifting 产量 × 惊喜质量 (借 Felt / Select-the-Unexpected / Awash / Ryan)
- **机制**: 维护一组 *sifting patterns*(背叛、复仇闭环、师徒反目、派系吞并、寒门逆袭…), 在事件流上跑**增量逻辑查询**, 匹配出"成链的好故事"。
- **fitness = Σ(成链好故事数 × 每条质量)**, 其中质量 = **统计不可能度**(StU: observed vs expected 频率, 越罕见越惊喜) × **因果链完整度**(Ryan 的 backward-chaining: 能从终局回溯到自洽起点) × 链长/跨度。
- **关键教训 (Tale-Spin, 1976)**: 角色各追私目标产出的极少是故事, 而是"无全局结构的问题求解轨迹"。**纯涌现 ≠ 故事**, 必须在模拟之上加"筛/导"层。
- **防 game**: 必须用"惊喜/罕见"加权, 否则进化会刷"高频但无聊"模式(人人天天结义); 加模式多样性约束 + pattern 库定期扩充。

### 1.2 辅信号: 关系网/派系冲突的图指标 (借 Ware 冲突四指标 + 字符网络/社区检测)
把五行生克社交建成动态加权图, 逐 tick 算:
- **极化/对立度**(社区检测模块度 + 跨阵营负边权重) · **balance**(对立两派势均否, 碾压不好看) · **directness**(敌对双方是否真有因果交锋) · **intensity/stakes**(冲突涉及的进阶/更替/资源赌注) · **网络变化率**(阵营隶属翻转量, 剧变=有戏)。
- **防 game**: 只奖"高冲突"会逼出永恒混战 → **必须配 resolution 项**(冲突要有阶段性化解), 否则曲线顶格也是无聊。要"张弛"不要"长期满负荷"。

### 1.3 开放度/抗停滞: 历史新颖度 (借 ASAL)
- ASAL 用 `min` 当前状态对**所有历史状态**的最大相似度 → 逼模拟持续产出"历史新颖"的局面。我们已有 embedding, 可把每章/事件链向量化, fitness 加一项"新章 vs 所有旧章的最大相似度越低越好" + 共存角色/派系/事件类型的熵。
- 这是对抗 Tierra/Avida 式**停滞**(新颖性枯竭)的根本招。

### 1.4 LLM 当裁判: 只作 tie-break 与校准, 不作唯一 fitness
- 系统级排序 LLM judge 还行(优于纯自动指标), 但**绝对分不稳、对 prompt 敏感、易被"辞藻戏剧化"骗**。
- 用法: ①②③ 打平时 tie-break; 拿少量 LLM 分回归校准符号指标; 强制结构化输出 + pairwise + 多采样取中位压偏置; 定期换 prompt/模型抽查是否被 game。

> **一句话**: 主 fitness = story-sifting 产量×惊喜(可验证、抗 game); 辅 = 派系图张力(含 balance/resolution); 加 ASAL 历史新颖度抗停滞; LLM 仅校准。

## 2. 更丰富的涌现底座 (光有 fitness 不够, 得先让世界有东西可涌现)

我们现在每 tick 只让 focus 轮转的少数角色行动, 非 focus 角色"冻结", 派系/关系网是被动标签。三类机制让世界本身长出复杂度:

### 2.1 资源/能量账本 → 生态自发涌现 (借 Tierra + Avida + Flow-Lenia)
- **机制**: 稀缺资源自发催生竞争、寄生、生态位分化。Tierra 程序抢 CPU → 涌现寄生体/抗寄生宿主; Avida **奖励积木式子任务** → 复杂功能经"垫脚石"涌现; Flow-Lenia **质量守恒** → 固定总量必须在斑图间分配 → 领地/竞争自发。
- **映射**: 给每个角色一个**资源池**(气运/声望/灵石/信众/人脉, 可耗竭、可掠夺、可再生), 动作消耗+产出资源, 进阶/收徒需资源阈值。**学 Flow-Lenia 质量守恒**: 全世界资源近似守恒 → 逼出零和竞争与此消彼长。
- **预期**: 派系消长、寄生型配角(食客/二五仔)、生态位分工(谋士/打手/掮客)**自发**出现, 不靠写死剧情。
- **代价**: 需防"赢家通吃"坍缩 → 加资源再生 + 反垄断衰减。

### 2.2 社交感知驱动的角色分工 → 派系做活 (借 Project Sid 最硬结论)
- **Sid 铁证**: 只有当 agent 能**感知他人目标并主动避免冗余**, 分工才涌现; **关掉社交感知 → 角色熵塌平、行为同质重复**。
- **映射**: 先验打分里加一项"**生态位互补/冲突**"分 —— 角色选动作前廉价读同派系/邻近角色近期目标画像, 对"填补本派系空白生态位(缺掌财/掌律/外交)"加分, 对"挤占已饱和生态位"减分。
- **预期**: 派系从同质打手集合 → 自发分化出钱权/律法/情报/打手等**功能角色**; 派系消长从"人数比拼"升级为"职能完整度比拼"(职能残缺者易被吞并) → 天然戏剧张力。
- **代价**: 多一次读邻居开销 → 限感知半径(只读同派系+相邻者, 防 N²)。

### 2.3 社会结构本身随世界生长 (借 Static-Sandboxes 协同演化 + Sid 制度涌现 + 命名游戏临界质量)
- 这是"模拟层自进化"的核心: 现在派系数/关系网密度是**预设恒定**的。引入由世界状态触发的**结构生长算子**:
  - **派系分裂/新生**: 内部生态位冲突分持续高 + 出现关系网局部稠密的小团体(对标命名游戏 **~20% 坚定少数临界质量**) → spawn 子派系叛离;
  - **派系消亡/合并**: 职能熵塌缩或核心尽殁 → 降为附庸或并入;
  - **制度涌现**: 多派系反复就同一资源冲突 → 触发"修规"事件(投票/盟约) → 生成一条**世界级新规则**写入先验, 后续打分都受其约束。
- **预期**: 派系数与关系网密度随 tick 自然涨落而非恒定, 世界**越跑越复杂**, 大事由结构张力内生。
- **代价**: 防失稳(无限分裂) → 设上限/熵预算 + 五行生克做稳定器(相生倾向合并、相克倾向分裂)。

### 2.4 全员轻量记忆 + 异步反思 (借 Generative Agents, 分层降本)
- focus 角色走完整每-tick LLM 反思+规划(现状不变); **非 focus 角色后台跑廉价异步反思** —— 不每 tick, 而是"被卷入事件重要性"累加超阈值才用小模型把零散经历抽成一条"关系判断"写回。
- **预期**: 非 focus 角色态度不再静止, 轮到当 focus 时立刻有"积怨/恩情"可演。**代价**: 后台 LLM 量增 → 便宜模型 + 事件去重 + 批处理(参照 AI Town 的 tick/LLM 解耦)。

## 3. 规则/参数自进化引擎 (怎么让上面这些自己变好)

分两个力度, 由轻到重:

### 3.1 轻: 把写死的模拟动力学变成可进化旋钮 (扩展现有 genome + MAP-Elites)
- 现在 genome.engine 只有 `priorWeight`。**扩展为一族模拟旋钮**: 资源再生率、稀缺度、冲突/张力累积速率、大事触发阈值、登场/陨落/更替率、生态位互补权重、结构生长强度、混沌边缘目标。
- core 引擎只读 `props.tuning.*`(已有 priorWeight 这个钩子, 范式现成、保持 genre 中立), 这些旋钮当通用数值乘子。
- 用现有 **MAP-Elites/QDAIF 存档**进化它们, fitness = 第 1 节的模拟层 fitness(而非作者层文本 fitness)。
- **这是最快见效、风险最低、复用最多现有基建的一步。**

### 3.2 重: 让 LLM 提议全新机制/规则 (借 OMNI-EPIC + Voyager + Ludi, 把"铁律"下沉到模拟层)
- 我们已有作者层的"铁律"提议→校验→准入流程(`constraints.ts`)。**把同一套流程下沉到模拟层**, 作用于"世界法则"而非文字:
  - LLM 产出**结构化规则对象**(新大事类型: 触发条件用现有状态字段、影响哪些派系/角色、产出何种状态变化), core 只认 schema 不认 genre 字面量;
  - 或 OMNI-EPIC 式让 LLM 写**狭窄签名的纯函数**(`propose(state)->bool` / `apply(state)->patch`)放受限沙箱(无 IO、白名单 API、超时、确定性种子)。
- **进化"规则形态"而非只调权重 (借 Ludi ludeme + 进化 CA 规则)**: 把五行生克/八字打分写成可进化的**表达式树/DSL**(节点=五行关系、十神、派系距离等已暴露算子), 进化的是结构(如新增"三合局触发群体增益""墓库改变生克方向"), 权重只是叶子。
- **三道自洽闸 (这是最该抄的部分, 防写崩)**:
  1. **静态闸**: 新规则引用的字段/动作必须都在 core 已暴露状态空间内, 否则拒(+ 铁律硬约束: 不能让已死角色行动等);
  2. **新颖闸**: RAG 检索历史规则/大事, 过近则丢(防刷同质);
  3. **影子模拟闸 (MCC 思想)**: 沙箱空跑 N tick, 触发频率须落 [下限,上限] 区间、且确实改变派系/五行分布; OMNI-EPIC 式编译报错**自动迭代 5 次**修正; 通过才"晋升"为正式机制(Voyager 技能库范式)。
- **事务化写入**: 新机制以 patch/diff 形式提交, 闸门过才 commit, 可回滚。
- **这是研究级、慢、有写崩风险, 但最接近"模拟器自己进化出新东西"。**

### 3.3 协同进化"世界难度曲线" (借 POET transfer + Voyager 自动课程)
- 把"大事/危机"当 POET 的环境, "派系/主角"当 agent, 维护多条世界线允许机制迁移; 按当前角色实力自动调挑战强度, 制造"垫脚石"式剧情升级(而非随机难度)。
- **MCC 闸**: 新危机必须有现存派系能"中等概率应对"才上场, 太碾压/太软都丢。复用我们已有的单写者锁/端口隔离做世界线隔离。

## 4. 混沌边缘的自我调节 (让世界长期"丰富但不崩")

### 4.1 临界控制器 (借 自组织临界 / Langton's λ)
- 复杂涌现只存在于"混沌边缘"。监控**事件雪崩规模分布**(一次冲突牵连多少角色/派系): 呈幂律即在临界。
- 太冷(万年无事) → 调高资源稀缺/矛盾注入; 太热(天天灭门、角色批量陨落) → 调高再生/降冲突增益。
- 把单个 priorWeight 旋钮**升级为一族自调旋钮的临界控制器**(负反馈: 太死加扰动, 太乱加五行稳定器)。

### 4.2 戏剧导演 = Drama Manager (借 Façade 张力弧 + RimWorld Cassandra + Awash + Bayesian surprise)
- 维护**目标张力弧**(Aristotelian: 升-高潮-落)+ 世界信念模型(对"接下来会发生什么"的概率先验)。
- 逐 tick 算: 当前张力(=第 1.2 节图指标) + **Bayesian surprise**(实际事件相对先验的 KL 散度)。
- **触发**: 当前张力 < 目标弧 + 持续 K tick 低惊喜(太可预测) → director 注入干预: 抬高某派系野心、降某关键角色保守项、投放大事/外部威胁。
- **关键 (Awash)**: 干预要"**顺水推舟**"已有的半成形 sifting 故事链, 而非凭空塞 —— 否则变 Randy Random 式廉价刺激, 读者麻木。
- **闭环**: 干预后必须由第 1 节 fitness 复核"这次干预真提升了成链好故事/张力起伏吗", 否则回滚。

## 5. 诚实评估: 哪些给质变, 哪些不给, 真实风险

**会带来质变的**:
- ✅ **资源账本 + 结构生长算子**: 从"角色在固定派系里打分选动作" → "生态/势力/制度本身会生灭" —— 这是定性的复杂度跃升(MODES complexity 意义上), 是真正的"世界在演化"。
- ✅ **story-sifting fitness**: 第一次让"有不有戏"变成可优化的客观信号 → 进化能朝它爬。这是把作者层进化的成功范式真正搬到模拟层的钥匙。
- ✅ **混沌边缘控制器**: 解决"要么万年太平要么天天灭门"的两极, 让世界稳定停在最高产戏区。

**多半不会单独带来质变的**:
- ⚠️ 只做 3.1(扩展旋钮 + MAP-Elites)而不做第 2 节的新机制 —— 那只是"把现有动力调得更准", 仍是同一概念空间内的量变(和作者层参数进化同性质)。**旋钮自进化必须配涌现底座才有质变**。
- ⚠️ LLM judge 当主 fitness —— 会被骗, 越进化越"假戏剧化"。

**真实风险 (全部研究都点到)**:
- **没人解决"自动越来越复杂且不崩"** —— 这是开放式进化的公开难题, 我们不会例外, 只能用临界控制器 + 闸门尽量逼近, 不能承诺无限开放。
- **纯涌现 ≠ 故事** (Tale-Spin 教训) —— 必须保留导演/策展层。
- **fitness 被 game** —— 主力必须是可验证符号指标, LLM 只校准。
- **规则自进化写崩世界** —— 必须沙箱 + 影子模拟 + 事务回滚 + 铁律硬约束。
- **成本** —— 全员反思/影子模拟/embedding 都烧 token, 需分层降本(便宜模型 + 批处理 + 采样)。

## 6. 分级落地路线 (建议)

| 阶段 | 内容 | 价值 | 风险 | 复用现有 |
|---|---|---|---|---|
| **T1** | 模拟层 fitness: story-sifting(pattern 库+增量匹配) + 派系图张力(balance/resolution) + ASAL 新颖度 | ★★★ 一切的前提 | 低 | events 日志、embedding |
| **T2** | 扩展 genome.engine 旋钮(资源再生/冲突率/大事阈值/更替/生态位权重/临界目标) + MAP-Elites 用 T1 fitness 进化 | ★★ 快见效 | 低 | genome、props.tuning 钩子、MAP-Elites 存档 |
| **T3** | 涌现底座: 资源账本(质量守恒) + 社交感知生态位分 + 结构生长算子(派系分裂/合并/制度) | ★★★ 真质变 | 中 | world-actor、pack 动力学(app/pack 层, core 中立) |
| **T4** | 混沌边缘控制器 + 戏剧导演(张力弧+Bayesian surprise+Awash 顺水推舟) | ★★★ 稳产戏 | 中 | director、event 系统 |
| **T5** | LLM 提议新机制/规则(铁律下沉到模拟层) + 表达式树进化打分规则 + 沙箱/影子模拟/MCC 闸 | ★★ 最激进 | 高 | constraints.ts 流程、沙箱 |

**推荐**: T1 → T2 先落地(可验证、低风险、复用最多), 拿到"模拟层 fitness 曲线"后再上 T3/T4(真正的质变在这), T5 作为研究级最后做。
