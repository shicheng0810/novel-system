# 长篇生成「结构一致性」能否系统性解决 · SOTA 综述 + 可交接实施规格
> 2026-06-12 · 深研(5 路并行检索 + 交叉核验)· 产出供 Claude CLI(Opus 4.8)执行
> 课题: 无状态 4 段/章流水线写 1000 章温润连载, 反复犯结构连续性 bug(物件改皮/可数不守恒/重复初遇/离驿重写/时序倒流/地理折返/伏笔不收), 文风指标测不到、生成端没预防。

---

## 0. 一句话结论
**可以系统性改进, 但路径与你手做出的判断高度一致, 文献几乎逐条印证。** 三条主轴:
1. **全上下文(FULLCTX)是死路, 且有机理**——不是调参失误。喂回自己的逐字旧文会激活 induction head 复制电路 → 套语爆炸是被预测的必然。
2. **窄带结构化状态(实体/物件表, 非全文)是唯一有验证的预防路**——多篇拿到可观一致性增益且**不**伤文风; 同质化恰恰只由"回喂生成的散文"引起, 喂"数据型状态"按构造避开。
3. **检测只能做成"高精度/半召回"的外部尺子, 不是零误杀闸**——依赖 coref+NLI; 你把 D20 降成分诊、把 revise 禁止改情节/时空, 都被证明是对的。

下面 Part A 综述、Part B 给 Opus 的实施规格、Part C 投资优先级。**先量发率再投资**的纪律全程保留。

## Part A · SOTA 综述(两侧均衡)

### A1. 为什么全上下文会退化(印证 FULLCTX 否决)
- **Lost in the Middle**(Liu 2023, TACL): 长上下文里中段事实检索准确率掉 >30%, 呈 U 形。埋在 6000 字中段的旧设定是**最不**被注意的。[arxiv 2307.03172]
- **NoLiMa**(2025): 12 个号称 128K+ 的模型, 到 32K 时 10–11 个跌破短上下文基线的 50%; **即使把无关 token 全 mask, 仍有 ≥7.9% 下降**——长度本身就是因果性退化, 剪枝无关也不够。[arxiv 2502.05167]
- **Context Rot**(Chroma 2025): 18 个前沿模型在远低于窗口上限处就开始变差。6000 字不是安全区, 是退化早坡。[trychroma.com/research/context-rot]
- **机理(最关键)**: induction heads 检测某 token 上次出现并复制其后续; 当该电路主导 logits 即"induction-head toxicity"→ 刚性复制 + 熵坍缩。**把自己 6000 字旧文逐字喂回 = 递给复制模板, 套语爆炸是机理预测的必然, 不是 tuning 偶然。**[Olsson 2022 arxiv 2209.11895; Repetition Neurons 2410.13497] 叠加解码层面 nucleus 退化(Holtzman 1904.09751)与"repeat-in-repeat-out"自强化(2310.10226)。
- **你的实测被发表工作确认并解释**: FULLCTX 否决正确, naive 长上下文对 1000 章一致性是死路。

### A2. 预防侧: 计划 + 结构化世界状态(你下一步的主路)
- **Plan-and-Write**(Yao AAAI'19): 先规划故事线再实现文本 → 更连贯/多样/不跑题。所有 plan-first 系统的根。[ojs.aaai.org/.../4726]
- **Re3**(Yang EMNLP'22): plan→draft→rerank→edit, 反复把结构化计划+当前状态重注入; 人评连贯+14%、前提相关+20%; **edit 步专门检测并改写与前文的矛盾**(一致性当离散修订, 非靠更好计划)。[2210.06774]
- **DOC**(Yang ACL'23): 细纲 + controller 强制段落贴纲, 比 Re3 +22.5% 情节连贯/+28.2% 纲相关/**+20.7% 趣味性**——更强控制**没有**同质化文风, 控制力与质量同升。[2212.10077]
- **DOME**(NAACL'25): 时序知识图谱"记忆增强模块", **上下文冲突减 87.61%** 且流畅/连贯/质量齐升——结构化(图)状态而非散文回忆驱动增益。[2025.naacl-long.63 / 2412.13575]
- **StoryState**(2026): 世界=可读可编辑的 JSON 状态(角色表+全局设定+逐景状态)做唯一真相源喂生成——**紧凑结构化状态而非全文**也能驱动一致且人可编辑的长文。[2602.01305·新源, 实施时复核]
- **MNEME 实体记忆**(2202.01709): 给小模型挂紧凑实体 KV 记忆 → 长程实体连贯**相对提升 57%**、人评一致性升。
- **多 agent + 故事 KG**(2508.03137): 双层记忆(长期只留最重要事实防主题漂 + 短期近纲)**明确因为"喂回一切会失败"**而选紧凑选择性状态。
- **ConStory/Lost in Stories**(2603.05890·新源): 基准把"事实&细节一致性"与"时间线&情节逻辑"标定为长文主导失效模式, 且**集中在叙事中段、高熵段**——正是窄带账本要覆盖的离散可枚举事实。
- **副作用核查**: 上述预防类**无一报告文风退化**(DOC 趣味升、DOME 流畅升)——所惧的同质化权衡在这些测量层级**不成立**。真实代价是(a)多 pass 推理成本与图更新/冲突检测开销;(b)**状态抽取误差——账本只和填它的 LLM 一样准, 而主导失效(时序/细节)正是账本必须抓对的**(第一风险, 见 Part B 风险节)。

### A3. 修订侧: agentic 草稿-批判-修订的边界(印证你的 revise 约束)
- **同模型自我批改, 无外部信号时可能更差**: intrinsic self-correction 有时**降低**表现(复用产生错误的同一知识)。[2310.01798]
- **Self-Refine**(2303.17651): 约 +20% 偏好增益, 但**集中在开放式质量/风格, 对需外部可验证正确性的任务最弱**——它磨表面, 不修硬事实。
- **Reflexion**(2303.11366): 靠言语反思改善, 但**需每轮外部 reward/test 信号**才能反思, 非独立自判——不给"矛盾在哪"的 oracle 就迁移不到一致性。
- **CoVe 链式验证**(2309.11495): draft → 生成**独立**核验问题(不看 draft 作答)→ 修订; **解耦**(无偏的独立 pass 验证)是胜过单 pass 自批的关键。
- **关键**: SOTA 模型**检测不出植入的情节漏洞, 且随故事变长更差**[FlawedFictions 2504.11900]; 且 **LLM 生成本身就引入结构错误**(LLM 续写比人原作多 >100% 情节漏洞)——**不受约束的 revise 是新 bug 之源, 不是中性编辑**。
- **LLM-as-judge 自偏**: 判官给自己的续写打更高分(GPT-4o/Claude 3.5 实测)[2508.06709]——**同一模型既当 reviser 又当一致性判官会虚高通过率**。
- **结论**: 结构/情节/时序一致性**靠同模型自我批改修不可靠**; 瓶颈在**判官不在改写器**。你"revise 禁止动情节/时空、只修表面语言"是**安全且正确**的设计。结构病要(a)用**外部、证据锚定**的验证器检(CoVe 式解耦), (b)在**计划/状态层**修(DOC 式), 不在散文 rewrite 层。

### A4. 检测侧: 能做成什么尺子(印证 D20 降级)
- **书级 coref 是前提且现仍弱**: SOTA 长文 coref 在整本小说上只 ~36 CoNLL-F1(短文 80+)。[BookCoref 2507.12075]
- **NLI 一致性**: SummaC 顶到 ~74.4% 平衡准确率[2022.tacl-1.10]; **QA 式**(QAFactEval 2112.08542)同档 ~70s%。
- **FactScore** 原子claim分解可 <2% 误差, 但**只对外部可靠知识源**(维基), 非小说内部自相矛盾。[2023.emnlp-main.741]
- **实体网格**(Barzilay-Lapata)建模连贯流, **分不出"同物改写 vs 两物"**——证实纯词法做不到。[J08-1001]
- **专用 LLM 判官 ConStory-Checker**(抽事实→配对矛盾→证据链): **F1 0.678 / 精度 0.884 / 召回 0.550**, 且**胜过两名职业标注者**(人 F1 0.281、漏 ~83% 植入错)。[2603.05890·新源]
- **LLM 判官天然保守**: 二元判断假阳 0–1/136(精度 ~98.9%、FPR ~0.9%), 但细粒度打分会崩。[galileo.ai]
- **直接回答你的核心问题**: "同物改写 vs 两物共享名词"的零误杀确定性检测**确实不可能**——它本质是 coref 判定(这些 mention 是否同指)叠 NLI 判定(属性是否相容)。组件均非零误差(coref ~36F1/NLI ~74%/最佳小说判官 88P-55R)。**"零误杀+全召回"不可达, 只能拿召回换精度。** → D20 降成 CLI 分诊、不入逐章 gate, 正确。
- **可达目标**: 高精度/半召回的尺子, 加权到**事实+时序、叙事中段、高熵段**; 它已**比人工初校更准更便宜**(F1 0.68 vs 0.28)——目标是"远胜且远便宜于人工初审", 不是 ground truth。

## Part B · 实施规格(交接 Opus 4.8 CLI agent)
> 纪律: 零随机/确定性优先; 动 longrun 模块级代码必跑产线 env canary(见 runbook); 三语料校准(病例+好章误杀0+全语料发率); 不赌 gen3, 并行 A/B 舰队验证; 先量发率再投资。**绝不开 FULLCTX, 绝不同模型自批改 rewrite 结构。**

### Phase 0 — 先造尺子(最高 ROI, 解锁后续一切)
打破 Goodhart 闭环: 没有与人眼对齐的结构度量, 好修法会被盲指标判"无差异"砍掉(SEG_LEDGER 的命运)、坏副作用看不见(FULLCTX)。
- **建一个外部、证据锚定的"一致性判官" CLI**(ConStory-Checker 范式, 离线逐章跑, 非逐章 gate): 对一章正文 →(1)抽实体/物件/时刻事实清单 →(2)配对找矛盾 →(3)每条判定附**正文引文证据链**。调成**高精度**(宁漏勿误, 阈值靠保守二元判断)。
- **判官模型选择(防自偏)**: **不要**用写章的同一 DeepSeek 实例当判官; 用不同模型/不同 prompt 角色(本机可调 Opus/别的)做解耦判断。CoVe 式: 判官不看"作者意图"只看正文。
- **首要用途 = 量发率**: 拿它扫现役 gen3 近 N 章 + killed 归档, 给每类结构 bug(物件改皮/可数/重复初遇/时序/伏笔不收)一个**逐章发率**。这是后续所有投资档的依据(容器漂移 0.3% 就是这么量出来"不值得建账本"的)。
- 产出: `app/consistency-judge.ts`(或并入 ledger 工具族) + 一份发率基线表写进本 audit 目录。
- 验收: 判官在炊烟 ch119 原稿命中其已知硬伤、在精修稿静默; 与人读抽样一致。

### Phase 1 — 窄带章内状态账本(真正的预防杠杆 · 你的开放贡献)
文献一致指向"结构化状态而非散文"; 而**无人 A/B 过"物件表账本 vs 全文"在一致性+文风双轴**——这正是你的新贡献点。
- **schema 化、数据型(读起来像表不像叙述)**, 跨段维护、喂给 4 段流水线**替代不了也不等于** FULLCTX:
  ```
  实体/物件 → { 材质, 计数, 所在/持有者, 是否在场, 上一动作, 首现段 }
  人物 → { 已出场?, 已遇过谁, 当前位置 }
  时刻 → 单调时辰游标
  ```
- **喂法(避开 A1 的退化)**: (a)**紧凑 schema, 非散文**——按构造避开 induction-head 复制(同质化只由回喂散文引起, 见 A2 副作用节 & Padmakumar-He 2309.05196); (b)**检索/关键词门控**(NovelAI Lorebook 式), 只把当前场景相关实体入 prompt; (c)**放 prompt 边缘**(首或尾, 避中段死区, Lost-in-the-Middle); (d)**每段后按离散状态转移更新**(RecurrentGPT 2305.13304 / MemGPT 2310.08560 分页), 不累积成长散文。
- **与现有 SEG_LEDGER 的关系**: 这是它的**精化窄带版**——SEG_LEDGER 当年喂的偏全量/被盲尺判无效; 现在只喂"物件清单+材质/计数+时刻", 并用 Phase-0 尺子(非盲指标)重判。
- **A/B 验证(铁律)**: 臂A=窄带账本开 / 臂B=现栈, 各 8 章并行(别碰 gen3), **双轴判读**: ①一致性(Phase-0 判官的各类发率↓) ②**副作用轴必量**——套语 d1c 率(lint-seams 现成)、节律均质度(段尾停顿/模板均一)。**只有 d1c 不炸且发率明显降才采纳。**

### Phase 2 — 解耦的结构验证-修订(只补 Phase 1 漏网)
- **修订只在状态/计划层, 不在散文层**: 判官(Phase 0)报出矛盾 → 修订器只被允许改**账本/大纲**或做**最小外科替换**(D13/D18 那种单 token/删一句已遵从的 issues 通道), **维持 revise 禁止改情节/时空的现状**(A3 证明这是安全的)。
- **绝不**让同模型自由 rewrite 情节修结构(FlawedFictions: 会注入新漏洞)。

### Phase 3 — 跨章/全局(1000 章弧的伏笔与实体连续)
- **故事 KG/状态 DB**(DOME 范式): 跨章维护实体-关系-伏笔状态, 专治"伏笔开了不收"(沈无尘断线类)与跨章实体漂。离线增量更新 + 冲突检测, 喂大纲/canon 注入, 不喂全文。
- 这是重投资, **排在 Phase 0/1 验证有效之后**, 且先用 Phase-0 尺子量"伏笔不收"的发率定档。

|---|---|---|---|

## 风险与诚实边界
1. **状态抽取误差是第一风险**: Phase 1/3 的账本只和填它的 LLM 一样准, 而主导失效(时序/细节)正是要抓对的。→ Phase 0 判官应反过来抽查账本自身。
2. **判官自偏**: 必须用与写章不同的模型/解耦 prompt, 否则通过率虚高(2508.06709)。
3. **新源待复核**: ConStory/StoryState/DOME 等 2025–2026 源(2603.05890/2602.01305/2412.13575)较新, 实施时复核具体数字; 但奠基性结论(Lost-in-Middle/induction heads/Re3/DOC/Self-Refine/CoVe/SummaC/entity-memory)均成熟可靠且交叉印证。
4. **开放缺口=你的贡献**: **无任何已发表工作直接 A/B 过"物件表账本 vs 全文"在一致性+文风双轴**——Phase 1 的双轴验证本身就是可发表的新结果。
